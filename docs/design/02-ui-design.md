# UI 设计规范

本文档定义 Blinko / 明眸 项目的 UI 设计规范，强调**精简设计**。

---

## 1. 设计核心：精简至上

### 1.1 设计哲学

> **"简单是终极的复杂"** - 达芬奇

- **最小化界面元素**：只显示必要的信息
- **减少视觉噪音**：使用一致的视觉语言
- **避免功能堆砌**：只实现核心功能
- **默认值优先**：减少用户配置负担

### 1.2 设计决策原则

在做出任何设计决策前，问自己：
1. 这个元素是否必需？
2. 能否用更简单的方式实现？
3. 能否删除某些部分？

**如果答案不确定，先不做。**

## 2. 视觉设计

### 2.1 颜色系统

**主色调**（精简到 3-4 种）：

- **主色**：#4A90E2（柔和蓝）- 用于主要操作和品牌
- **辅助色**：#50C878（健康绿）- 用于正常状态
- **警告色**：#FFB84D（温暖橙）- 用于提醒状态
- **错误色**：#FF6B6B（柔和红）- 用于需要休息状态
- **中性色**：
  - 文本：#333333（深灰）
  - 次要文本：#666666（中灰）
  - 边框：#E0E0E0（浅灰）
  - 背景：#FFFFFF（白色）/ #F5F5F5（浅灰）

**使用原则**：
- 主色最多使用 2 种
- 功能色（状态色）使用 2-3 种
- 避免使用过多的颜色

### 2.2 字体系统

**字体选择**：
- **中文**：思源黑体 / 微软雅黑（系统默认）
- **英文**：Inter / Roboto

**字号层级**（精简到 4 级）：
- **标题**：18px（主标题）
- **副标题**：16px（卡片标题）
- **正文**：14px（主要内容）
- **辅助文字**：12px（次要信息）

**字重**：
- 正常：400
- 中等：500（强调）
- 粗体：600（标题）

### 2.3 间距系统

**间距单位**（8px 基准）：
- 4px（0.25rem）- 微小间距
- 8px（0.5rem）- 小间距
- 16px（1rem）- 标准间距
- 24px（1.5rem）- 中等间距
- 32px（2rem）- 大间距

**使用原则**：
- 使用一致的间距单位
- 组件内部使用小间距（8px）
- 组件之间使用标准间距（16px）
- 区块之间使用大间距（24-32px）

### 2.4 圆角系统

**圆角大小**（精简到 3 级）：
- **小**：4px - 按钮、输入框
- **中**：8px - 卡片
- **大**：12px - 模态框

## 3. 组件设计

### 3.1 按钮

**类型**（精简到 2 种）：
- **主要按钮**：使用主色，用于主要操作
- **次要按钮**：使用中性色，用于次要操作

**尺寸**（精简到 2 种）：
- **标准**：高度 36px，用于一般场景
- **小**：高度 28px，用于紧凑场景

**设计原则**：
- 避免过多的按钮类型
- 使用清晰的标签文本
- 避免图标+文字的复杂组合

### 3.2 卡片

**设计规范**：
- 白色背景
- 8px 圆角
- 轻微阴影（0 2px 8px rgba(0,0,0,0.1)）
- 内边距：16px

**使用场景**：
- 数据展示
- 功能模块
- 统计信息

**避免**：
- 过多的装饰
- 复杂的布局
- 过多的信息

### 3.3 输入框

**设计规范**：
- 高度：36px
- 边框：1px solid #E0E0E0
- 圆角：4px
- 内边距：8px 12px

**状态**：
- **默认**：灰色边框
- **聚焦**：主色边框
- **错误**：错误色边框

### 3.4 图标

**设计原则**：
- 使用线性图标风格（一致）
- 尺寸：16px / 20px / 24px
- 颜色：使用文本颜色或主色
- 避免过多的图标样式

## 4. 布局设计

### 4.1 主面板布局

**设计原则**：
- 单列布局（避免多列）
- 上下滚动（不横向滚动）
- 卡片式布局（清晰分组）

**布局结构**：
```
┌─────────────────────────┐
│  顶部栏（标题）          │  高度：48px
├─────────────────────────┤
│  标签页（今日/历史/设置）│  高度：40px
├─────────────────────────┤
│  内容区域（滚动）        │
│  ┌─────────────┐        │
│  │  数据卡片   │        │
│  └─────────────┘        │
│  ┌─────────────┐        │
│  │  简单图表   │        │
│  └─────────────┘        │
└─────────────────────────┘
```

**窗口尺寸**：
- 默认：400px × 600px
- 最小：300px × 400px
- 最大：500px × 800px（不推荐）

### 4.2 提醒弹窗布局

**设计原则**：
- 居中显示
- 简洁清晰
- 操作明确

**布局结构**：
```
┌─────────────────────┐
│  [Blinko 图标]      │  高度：60px
│  该休息了！          │
│  看看远方 20 秒吧   │
│                     │
│  [开始休息] [跳过]   │  高度：40px
└─────────────────────┘
  宽度：300px
```

## 5. 交互设计

### 5.1 交互原则

- **减少点击**：目标操作不超过 2 步
- **清晰反馈**：每个操作都有视觉反馈
- **容错设计**：允许用户撤销或取消
- **渐进式展示**：需要时再显示详细信息

### 5.2 动画设计

**动画原则**：
- 轻量级：使用 CSS 动画，避免复杂库
- 有意义：传达状态变化
- 不打扰：不干扰用户注意力

**动画类型**（精简到 3 种）：
- **淡入淡出**：200ms - 用于显示/隐藏
- **轻微缩放**：150ms - 用于按钮点击
- **平滑移动**：300ms - 用于页面切换

**避免**：
- 过多的动画效果
- 复杂的动画序列
- 影响性能的动画

### 5.3 反馈设计

**反馈类型**：
- **成功**：绿色提示，短暂显示
- **警告**：橙色提示，短暂显示
- **错误**：红色提示，需要用户操作
- **加载**：简单的加载指示器

**设计原则**：
- 反馈及时（< 100ms）
- 信息清晰
- 不打断用户操作

## 6. 响应式设计

### 6.1 窗口适配

**原则**：
- 固定最小宽度（300px）
- 内容自适应高度
- 避免横向滚动

### 6.2 高 DPI 适配

**原则**：
- 使用矢量图标（SVG）
- 使用相对单位（rem/em）
- 测试不同 DPI 设置

---

## 7. 无障碍设计

### 7.1 基本原则

- **颜色对比度**：符合 WCAG AA 标准（4.5:1）
- **键盘导航**：支持 Tab 键导航
- **屏幕阅读器**：关键元素添加 aria 标签
- **字体大小**：支持系统字体大小设置

### 7.2 实现要求

- 使用语义化 HTML
- 添加适当的 aria 属性
- 确保焦点可见
- 提供替代文本

---

## 8. 设计检查清单

在设计任何 UI 组件时，检查：

- [ ] 是否只包含必要元素？
- [ ] 是否减少了视觉噪音？
- [ ] 是否使用了一致的视觉语言？
- [ ] 是否提供了清晰的反馈？
- [ ] 是否易于理解和操作？
- [ ] 是否避免了过度设计？
- [ ] 是否符合无障碍标准？

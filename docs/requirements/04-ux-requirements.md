# 用户体验需求文档

本文档详细描述 Blinko / 明眸 的 UI/UX 设计原则、交互规范和品牌风格。

---

## 1. 设计原则

### 1.1 核心原则
- **轻量不打扰**：最小化界面侵入，专注于系统托盘和通知
- **友好且智能**：使用 Blinko 角色化设计，增加亲和力
- **文化内核**：中文场景体现"明眸"文化底蕴，英文场景体现科技萌
- **隐私透明**：清晰告知用户数据使用方式

### 1.2 交互原则
- **非阻塞式**：提醒不打断用户工作流程
- **可快速响应**：所有操作响应时间 < 100ms
- **容错设计**：允许用户跳过、延迟、关闭提醒
- **渐进式引导**：首次使用提供简单引导

---

## 2. 品牌视觉设计

### 2.1 Logo 设计概念
- **形象**：半闭眼的猫头鹰/熊猫/机器人
  - 猫头鹰：象征智慧、专注（符合"明眸"）
  - 熊猫：亲切、可爱（符合"科技萌"）
  - 机器人：现代、AI 感
- **眼睛特征**：有"明眸"光泽，可动态眨眼
- **字体**：英文 "Blinko" 使用圆润、友好字体
- **颜色**：主色调待定（建议：柔和蓝/绿，代表健康）

### 2.2 色彩方案

#### 主色调（待定）
- **主色**：#4A90E2（柔和蓝）- 代表信任、科技
- **辅助色**：#50C878（健康绿）- 代表健康、活力
- **警告色**：#FFB84D（温暖橙）- 代表提醒、注意
- **错误色**：#FF6B6B（柔和红）- 代表需要休息

#### 状态颜色
- 🟢 **正常**：健康绿 (#50C878)
- 🟡 **久坐**：温暖橙 (#FFB84D)
- 🔴 **需休息**：柔和红 (#FF6B6B)

### 2.3 字体规范
- **中文**：思源黑体 / 微软雅黑（系统默认）
- **英文**：Inter / Roboto（现代无衬线字体）
- **字号**：
  - 标题：16-20px
  - 正文：14px
  - 辅助文字：12px

## 3. 界面设计规范

### 3.1 主面板设计

#### 布局结构
```
┌─────────────────────────────────┐
│  [Blinko Logo]  Blinko / 明眸   │  ← 顶部栏
├─────────────────────────────────┤
│  [今日] [历史] [设置]           │  ← 标签页
├─────────────────────────────────┤
│                                 │
│     今日概览卡片区域            │
│                                 │
│  ┌──────────┐  ┌──────────┐   │
│  │ 提醒次数 │  │ 休息时长 │   │
│  └──────────┘  └──────────┘   │
│                                 │
│     简单图表区域                │
│                                 │
└─────────────────────────────────┘
```

#### 设计要点
- **最小化设计**：默认窗口大小 400x600px
- **卡片式布局**：使用圆角卡片展示数据
- **简洁图标**：使用线性图标风格
- **留白充足**：避免拥挤感

### 3.2 提醒弹窗设计

#### 布局结构
```
┌─────────────────────────────┐
│  [Blinko 动画]              │
│  该休息了！                 │
│  看看远方 20 秒吧 👁️       │
│                             │
│  [开始休息] [跳过] [延迟]   │
└─────────────────────────────┘
```

#### 设计要点
- **居中显示**：不遮挡用户主要工作区域
- **动画反馈**：Blinko 眨眼动画增加趣味性
- **清晰按钮**：主要操作按钮突出显示
- **自动关闭**：5 秒后自动最小化（可配置）

### 3.3 系统托盘设计

#### 图标设计
- **正常状态**：绿色圆点 + Blinko 图标轮廓
- **久坐状态**：橙色圆点 + Blinko 图标轮廓
- **需休息状态**：红色圆点 + Blinko 图标轮廓（可闪烁）

#### 右键菜单
```
┌─────────────────┐
│ 打开主面板      │
│ 今日统计        │
│ ─────────────   │
│ 设置            │
│ 暂停提醒        │
│ ─────────────   │

└─────────────────┘
```

## 4. 交互流程设计

### 4.1 首次启动流程
1. **欢迎界面**
   - 显示 Blinko Logo 和欢迎语
   - "欢迎使用 Blinko / 明眸"
   - "让我们开始保护你的眼睛健康吧"

2. **权限请求**
   - 摄像头权限（可选）
   - 通知权限（必需）
   - 清晰说明用途

3. **快速设置**
   - 提醒间隔选择（20 分钟默认）
   - 是否启用眨眼检测
   - 是否启用坐姿检测

4. **完成引导**
   - "设置完成！Blinko 已开始工作"
   - 应用最小化到托盘

### 4.2 提醒交互流程
1. **触发提醒**
   - 系统托盘图标闪烁
   - 桌面通知显示（不阻塞）

2. **用户响应**
   - **点击通知** → 打开提醒弹窗
   - **忽略通知** → 5 分钟后再次提醒（可配置）

3. **弹窗操作**
   - **开始休息** → 显示 20 秒倒计时，完成后关闭
   - **跳过** → 记录跳过，关闭弹窗
   - **延迟** → 5 分钟后再次提醒

### 4.3 设置交互流程
1. **打开设置**
   - 从主面板或托盘菜单进入
   - 分类显示设置项

2. **修改设置**
   - 实时保存（无需确认）
   - 部分设置需重启生效（提示）

3. **保存反馈**
   - 轻微动画反馈
   - 成功提示（可选）

## 5. 动画与反馈

### 5.1 动画原则
- **轻量级**：使用 CSS 动画，避免复杂动画库
- **有意义**：动画传达状态变化或操作反馈
- **不打扰**：动画不干扰用户注意力

### 5.2 动画场景
- **Blinko 眨眼**：提醒触发时播放
- **图标闪烁**：托盘图标状态变化时
- **按钮点击**：轻微缩放反馈
- **数据更新**：数字递增动画

### 5.3 声音反馈
- **提醒声音**：轻柔"滴"声或鸟鸣（可选）
- **操作反馈**：按钮点击声（可选，默认关闭）
- **音量控制**：用户可调节或关闭

## 6. 多语言适配

### 6.1 文案风格
- **中文**：
  - 文雅、文化感："明眸善睐，坐如钟"
  - 亲切但不失专业："该休息了！看看远方 20 秒吧"
- **英文**：
  - 友好、现代："Time to rest! Look 20 feet away for 20 seconds"
  - 简洁明了："Blinko watches, so you can rest."

### 6.2 界面适配
- 文字长度：中英文长度差异较大，需预留空间
- 字体渲染：确保两种语言都清晰易读
- 图标标识：使用图标辅助，减少文字依赖

---

## 7. 响应式设计

### 7.1 窗口大小
- **最小尺寸**：300x400px
- **默认尺寸**：400x600px
- **最大尺寸**：600x800px（可选）

### 7.2 适配不同屏幕
- 高 DPI 屏幕：使用矢量图标和 SVG
- 不同分辨率：使用相对单位（rem/em）

---

## 8. 可访问性

### 8.1 无障碍设计
- 键盘导航：支持 Tab 键导航
- 屏幕阅读器：关键元素添加 aria 标签
- 颜色对比度：符合 WCAG AA 标准

### 8.2 用户偏好
- 字体大小：支持系统字体大小设置
- 动画偏好：支持减少动画（系统设置）

---

## 9. 错误处理

### 9.1 错误提示
- **友好提示**：使用 Blinko 角色化语言
- **清晰说明**：告知用户问题和解决方案
- **操作引导**：提供明确的下一步操作

### 9.2 错误场景
- 摄像头无法访问：提示用户检查权限
- 检测失败：提示用户调整光线或位置
- 数据加载失败：提示用户重试

---

## 10. 启动动画

### 10.1 启动屏设计
- **Blinko Logo**：居中显示
- **加载动画**：Blinko 眨眼动画
- **加载文字**："明眸正在启动..." / "Blinko is starting..."

### 10.2 加载反馈
- 显示加载进度（如需要）
- 首次初始化显示进度条（tqdm 风格）
